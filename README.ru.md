# Сборка Docker‑образа Qwen‑Code

Воспроизводимый Docker‑образ для **Qwen‑Code**, в который включены все необходимые
утилиты (kubectl, helm, yq, jq, node и др.) и удобный entry‑point для записи
интерактивных сессий Qwen.

## Структура репозитория

```
.
├── packer.pkr.hcl            # шаблон Packer (HCL) – собирает образ
├── variables.pkr.hcl          # все переменные версий и переключатели функций
├── entrypoint.sh              # оболочка, записывает сессию asciinema,
│                               # копирует JSON‑лог и создает текстовый лог.
├── copy_session_log.sh        # (встроен в entrypoint.sh, теперь не нужен)
├── build_image.sh             # скрипт‑обёртка, запускает `packer build`
│                               # с опциональными флагами `--no-*`.
├── run.sh                     # стартует готовый образ в tmux‑сессии.
├── .gitignore                 # файлы/каталоги, игнорируемые Git‑ом.
├── .qwenignore                # файлы, игнорируемые Qwen‑Code tooling.
└── README.md / README.ru.md   # эти документы.
```

### Ключевые файлы

| Файл               | Описание |
|--------------------|----------|
| **packer.pkr.hcl** | Описывает шаги сборки Docker‑образа через Packer. Динамические provisioner‑ы копируют пользовательские сертификаты, `debian.sources` и необязательные бинарники (kubectl, helm, yq, jq). |
| **variables.pkr.hcl** | Содержит версии инструментов (`kubectl_version`, `helm_version`, `yq_version`, `jq_version`) и булевые флаги (`copy_certs`, `copy_sources`, `copy_kubectl`, `copy_helm`, `copy_yq`, `copy_jq`). |
| **entrypoint.sh** | Запускает `asciinema` для записи сессии, после завершения копирует сгенерированный `logs.json` в `<session>.json` и конвертирует файл `.cast` в читаемый текстовый `<session>.log`. |
| **build_image.sh** | Обёртка над `packer build`. Флаги `--no-certs`, `--no-sources`, `--no-kubectl`, `--no-helm`, `--no-yq`, `--no-jq` отключают соответствующие шаги. |
| **run.sh** | Запускает готовый образ (`qwen-code:latest`) в tmux‑сессии, монтируя текущий проект и конфигурацию пользователя из `~/.qwen`. При отсутствии образа автоматически вызывает сборку. |
| **.gitignore** | Исключает из репозитория артефакты сборки, кеши и временные файлы. |
| **.qwenignore** | Инструктирует Qwen‑Code игнорировать временные сессионные файлы (`.cast`, `.log`, `.json`) и каталоги (`.qwen/tmp`, `.qwen/debug`). |

## Сборка образа

```bash
# Сделать скрипт исполняемым (делается автоматически)
chmod +x build_image.sh

# Сборка со всеми опциями (по умолчанию)
./build_image.sh

# Пример: собрать без helm и jq
./build_image.sh --no-helm --no-jq
```

Скрипт запускает Packer, который:
1. Берёт базовый образ `debian:13.3-slim` (можно изменить через переменную `base_image`).
2. При необходимости копирует пользовательские сертификаты и `debian.sources`.
3. Устанавливает системные зависимости.
4. Скачивает указанные версии `kubectl`, `helm`, `yq`, `jq` прямо внутри контейнера.
5. Устанавливает `nodejs` и CLI‑утилиту Qwen‑Code.
6. Копирует `entrypoint.sh` (с встроенной логикой) в `/usr/local/bin`.
7. Тегирует полученный образ как `qwen-code:latest`.

## Запуск контейнера

```bash
chmod +x run.sh
./run.sh   # или ./run.sh /path/to/project
```

`run.sh`:

* Проверяет наличие Docker‑образа, при отсутствии собирает его (`build.sh`).
* Создаёт в проекте директорию `.qwen/sessions` с правами записи.
* Запускает контейнер от имени пользователя `agent` внутри tmux‑сессии.
* Монтирует ваш проект и все нужные каталоги из `~/.qwen` (skills, sessions, tmp, debug, …).
* Внутри контейнера `entrypoint.sh` записывает сессию, сохраняет JSON‑лог и читаемый `.log` файл.

### Управление контейнером

| Действие                | Команда |
|-------------------------|----------|
| Отключиться от tmux    | `Ctrl‑B D` |
| Подключиться вновь      | `tmux attach -t qwen‑<имя‑проекта>` |
| Убить tmux‑сессию       | `tmux kill-session -t qwen‑<имя‑проекта>` |
| Посмотреть логи контейнера (debug) | `docker logs <имя‑контейнера>` |
| Удалить остановленный контейнер | `docker rm <имя‑контейнера>` |

## Логи сессий

* **JSON‑лог** – `<проект>/.qwen/sessions/<timestamp>.json` (исходный `logs.json`).
* **Текстовый лог** – `<проект>/.qwen/sessions/<timestamp>.log` (результат `asciinema cat`).
* **Файл‑запись** – `<проект>/.qwen/sessions/<timestamp>.cast` (двоичный запись, сохраняется для возможного воспроизведения).

## Настройка

* Параметры образа и версии утилит задаются в `variables.pkr.hcl`.
* Чтобы добавить свои сертификаты, разместите их в каталоге `ca-certificates/` и включите флаг `copy_certs`.
* Чтобы использовать собственный `debian.sources`, поместите файл в корень проекта и включите `copy_sources`.
* При необходимости расширьте `entrypoint.sh` – он уже исполняется от пользователя `agent` внутри контейнера.
